The Node.js SDK (`simplex-chat` on npm) gives you two levels of control. The high-level `bot.run()` API handles profile creation, address setup, and event dispatch in a single declarative call. The lower-level `api.ChatApi` class exposes every API command and event — for bots that need fine-grained control over connections, groups, files, or multiple user profiles.

## Prerequisites

- Node.js 18 or later
- The `simplex-chat` CLI binary available in `PATH` (or specify the path explicitly in `dbOpts`)

## Install

```sh
npm install simplex-chat
```

For the PostgreSQL backend on Linux x86_64:

```sh
npm install simplex-chat --simplex_backend=postgres
```

You can also persist the backend choice in `.npmrc`:

```ini
simplex_backend=postgres
```

## Quick start: echo bot

<Steps>
  <Step title="Create bot.js">
    ```javascript bot.js
    const {bot} = require('simplex-chat')

    ;(async () => {
      const [chat] = await bot.run({
        profile: {displayName: 'Echo Bot', fullName: ''},
        dbOpts: {type: 'sqlite', filePrefix: './echo_bot'},
        onMessage: async (ci, content) => {
          await chat.apiSendTextReply(ci, content.text)
        }
      })
    })()
    ```
  </Step>
  <Step title="Start the bot">
    ```sh
    node bot.js
    ```

    On first run, the native `libsimplex` library is downloaded and verified. On macOS this may take 20–30 seconds for Gatekeeper verification.
  </Step>
  <Step title="Connect to your bot">
    The bot prints its SimpleX contact address to stdout on startup:

    ```
    Bot address: simplex:/contact#...
    ```

    Paste this address into any SimpleX Chat app to connect and start chatting.
  </Step>
</Steps>

## Database configuration

The `dbOpts` field accepts a `DbConfig` discriminated union — choose the variant that matches the backend you installed:

```typescript
// SQLite (default) — works on all platforms
{type: 'sqlite', filePrefix: './bot-data/bot'}
// optional: encryptionKey: 'sqlcipher-passphrase'

// PostgreSQL — Linux x86_64 only
{type: 'postgres', connectionString: 'postgresql://user:pass@localhost/botdb'}
// optional: schemaPrefix: 'bot_'  (defaults to 'simplex_v1')
```

PostgreSQL prerequisites:
- `libpq5` must be installed (`apt install libpq5` on Debian/Ubuntu)
- A running PostgreSQL server reachable via the connection string

## Handling events

`bot.run()` accepts several options to control connection behaviour and message handling:

```javascript
const {bot} = require('simplex-chat')

;(async () => {
  const [chat] = await bot.run({
    profile: {displayName: 'Help Bot', fullName: 'Help Bot v1'},
    dbOpts: {type: 'sqlite', filePrefix: './help_bot'},
    options: {
      addressSettings: {
        welcomeMessage: 'Welcome! Type /help to see available commands.',
      },
    },
    onMessage: async (ci, content) => {
      const text = content?.text ?? ''
      if (text === '/help') {
        await chat.apiSendTextReply(ci, 'Available commands:\n/help — show this message\n/about — about this bot')
      } else if (text === '/about') {
        await chat.apiSendTextReply(ci, 'Help Bot v1 — built with the SimpleX Node.js SDK.')
      } else {
        await chat.apiSendTextReply(ci, `Unknown command. Type /help for a list of commands.`)
      }
    },
  })
})()
```

The `onMessage` callback receives two arguments:

| Argument | Type | Description |
|---|---|---|
| `ci` | `AChatItem` | The full chat item, including `chatInfo` and `chatItem.meta` |
| `content` | `MsgContent` | Shorthand for `ci.chatItem.content.msgContent` — the message payload |

`bot.run()` returns `[chat, user, address]`. Use `chat.apiSendTextReply(ci, text)` to reply to an incoming message (quoting it), or `chat.apiSendTextMessage(chatInfo, text)` to send without quoting.

## Squaring bot: using ChatApi directly

For full control, use `api.ChatApi` from `simplex-chat`. This is the underlying client used by `bot.run()` internally. It is event-driven — subscribe handlers with `chat.on(...)` before calling `chat.startChat()`:

```javascript
const {api} = require('simplex-chat')

;(async () => {
  const chat = await api.ChatApi.init({type: 'sqlite', filePrefix: './squaring_bot'})

  chat.on({
    contactConnected: async ({contact}) => {
      console.log(`${contact.profile.displayName} connected`)
      await chat.apiSendTextMessage(
        {type: 'direct', contact},
        'Hello! Send me a number and I will square it.'
      )
    },
    newChatItems: async ({chatItems}) => {
      for (const {chatInfo, chatItem} of chatItems) {
        if (chatInfo.type !== 'direct') continue
        if (chatItem.content.type !== 'rcvMsgContent') continue
        const msg = chatItem.content.msgContent?.text
        if (msg) {
          const n = +msg
          const reply =
            typeof n === 'number' && !isNaN(n)
              ? `${n} * ${n} = ${n * n}`
              : 'That is not a number.'
          await chat.apiSendTextMessage(chatInfo, reply)
        }
      }
    },
  })

  const user = await chat.apiGetActiveUser()
  if (!user) throw new Error('No active user — create one via the CLI first')
  console.log(`Bot: ${user.profile.displayName}`)

  await chat.startChat()

  const userId = user.userId
  const address =
    (await chat.apiGetUserAddress(userId)) ||
    (await (async () => { await chat.apiCreateUserAddress(userId); return chat.apiGetUserAddress(userId) })())
  console.log(`Address:`, address?.connLinkContact.connFullLink)
})()
```

<Tip>
For advanced use cases — handling file events, group invitations, custom command routing, or managing multiple user profiles — use `api.ChatApi` directly. It exposes typed wrappers for every WebSocket API command. See the [Commands reference](/bots/api-commands) and [Events reference](/bots/api-events) for the full list.
</Tip>
